Choose the events and dimensions that you want to track.

Events

Any event name used by Brightcove player's analytics-beacon event can be used, but the following have special handling in the GTM plugin.

Type Event Description
Player Load player_load Player has been loaded and is operational
Video Impression video_impression Video has been loaded into a player and is ready for playback
Video View video_view Video has started playing back either through auto-play or user interaction
Video Engagement video_engagement A user watches a range of seconds of a video's timeline. This event is a heartbeat for tracking video engagement and will likely be sent many times during playback, depending on the user interaction and the length of the video.
Play Request play_request Playback is initiated either by the user expressly clicking the play button, or automatically when the platform triggers playback in an auto-play scenario
Ad Start ad_start Control is handed over to an advertising agent by the playback platform
Ad End ad_end Control is handed back from the advertising agent to the playback platform
Error error A fatal error has occurred which disrupts the playback experience
Video Complete video_complete Video playback finishes
Please note that the events listed in the table above must be created as triggers.

Dimensions

The following dimensions will be sent for all events(except error)

Type Event Description
Account ID bcvideo_account Video Cloud Studio account ID
Playback location bcvideo_destination Playback location
Brightcove Player version bcvideo_platform_version Version of the player
Brightcove Player ID bcvideo_player ID associated with the player
Brightcove Player name bcvideo_player_name Name associated with the player
Session bcvideo_session Session associated with the player
Video ID bcvideo_video ID associated with the video
Video name bcvideo_name Name associated with the video
List of tags bcvideo_tags List of tags associated with the video
