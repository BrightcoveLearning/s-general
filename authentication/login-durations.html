The token durations discussed in this document are only applicable to Beacon native (OAuth) authentication. If you use third party OIDC authentication, you need to contact that provider for information on implementation and duration settings.

The viewer login duration is dependent on the login token and refresh token. Here are the rules for their duration:

  • The login token is good for one week.
  • The refresh token is good for one month.

Token definitions

If authentication is required, the viewer must use their login credentials to use the Beacon app. The question is then how long is the duration of successful login valid. The answer to this question is controlled by the duration of the login and refresh tokens.

  • The login token duration sets how long the viewer can use the app without re-authenticating. For instance, if the login duration is five days, every five days the viewer will have to use credentials to login again.
  • As a courtesy to viewers, you may wish to not force them to login too frequently, but still insure security. This is where a refresh token can be used. The use of the refresh token is as follows:
    • The viewer's login token expires or somehow becomes invalid.
    • The authorization server will use a valid refresh token to allow the viewer to obtain a new login token.
    • The viewer gets a new login token and can continue to use the Beacon app.

    Of course, at some point the refresh token will also expire and the viewer will have to use credentials to log in again.
