Introduction

The Roles as Rights feature ensures that content metadata is only viewable by viewers whose role matches the role set on the content.

With this feature, you can present specific content to viewers based on their assigned roles. The following content types can be shown or hidden based on the viewer's role:

  • Movies
  • Programs
  • Seasons
  • Series
  • Episodes
  • Channels
  • Live Events
  • Playlists
  • Pages

In the Beacon CMS, you are able to create roles and assign them to each each content type listed above.

You will use your OIDC roles claim URL to associate viewers with roles. OpenID Connect (OIDC) is an authentication protocol used to verify the identity of a user to a client service.

For details about using OIDC, see the OIDC Authentication Configuration and Testing document.

Adding a role

You can add roles in the Brightcove Beacon CMS.

  1. Login to the Brightcove Beacon CMS and click .

  2. In the left navigation, select Roles.

  3. Click Add new role.

  4. In the Create New Role dialog, enter a name for the role and click Save.

  5. To edit an existing role, click it's corresponding role ID.

  6. In the Edit role dialog, update the role name and click Update.

    Adding/editing Roles in Beacon does not propagate to your LDAP (Lightweight Directory Access Protocol). You must make role changes in both the Beacon CMS and your LDAP, and make sure that they match.

Assigning roles to content

You will use the Brightcove Beacon CMS to assign roles to series, seasons, channels, live events, playlists, and pages. These content types use a similar field for entering roles.

Let's walk through the steps for adding roles to a series.

  1. In the top navigation, select Programs. In the left navigation, select Series.

  2. In the Edit series screen, select Rights & Scheduling.

  3. Click Select/Unselect All Roles to select all available roles.

  4. Or, click inside the Asset roles input area and select a role from the list of available roles.
